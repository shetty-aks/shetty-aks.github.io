Google Summer of Code with Mozilla

Skills into action

HTML CSS Javascript Usablity Testing

Need and Summary

The project involves the redesign of Firefox Reader as well as working on the functionality related issues the current reader is facing. The redesign of the Reader has to be done so that it complies the Photon Design System and also offers a better user experience to it's users. Thus, using the knowledge of JavaScript, CSS, and web experience, the project aims at revamping the Reader View considering all the three reader modes ie. Light Mode, Dark Mode and Sepia Mode to enhance the reader's reading experience.

Research

The issues in the present Reader mode on analyzing the various bugs filled could be grouped into the following-
1. Design
2. Refactor the Code
3. Improove the accessiblty of the code.

Implementation

1. Redesigned the reader mode to Comply with the latest standards.
2. Integrated Keyboard Shortcuts with the Newly designed UI.
3. Refcatored the code in the process.

Need and Summary

The internship role involed two major tasks one was to improove the usablity of the existing application from the feedback the company had recieved so far. Secondly, the company's website had to be redesigned to carter to it's expanded client base.

Research

For the first task, the following issues were analyzed-
1. As the application was a smart compliance assistant, there was a lot of text going around the application. Thus, it was important to improove the readablity of this text for the user.
2. The existing UI bugs (mostly bootstrap related) needed to be solved
3. As the company had expanded, there was a need for the complicance system to get a name and chacter that would help them popularize it further.

Implementation

1. The readablity was improoved by combining knowledge of CSS and phycology of human reading.
2. The prevalent bugs were solved.
3. The smart bot was name "MICA" and was made the highlight of the newly designed webpage along with how it could help making compliance easier.
